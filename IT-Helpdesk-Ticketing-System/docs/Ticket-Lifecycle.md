# Ticket Lifecycle

## 1. Intake
Collect the user's name, contact information, device information, issue description, affected application, and urgency.

## 2. Categorization
Assign the correct issue category such as password reset, software installation, network issue, printer issue, or hardware issue.

## 3. Prioritization
Determine priority using impact and urgency. Assign SLA response and resolution targets.

## 4. Assignment
Assign the ticket to the correct support tier or technician.

## 5. Troubleshooting
Use structured troubleshooting:
- Confirm the issue
- Reproduce the problem when possible
- Check recent changes
- Apply known fixes
- Test with the user

## 6. Resolution
Apply the fix and verify that the user's issue is resolved.

## 7. Documentation
Record symptoms, troubleshooting steps, root cause, resolution, and time spent.

## 8. Closure
Confirm user acceptance, mark the ticket closed, and link any related knowledge base article.
