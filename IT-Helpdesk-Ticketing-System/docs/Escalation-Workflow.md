# Escalation Workflow

## Tier 1 Support Responsibilities
- Intake and categorize ticket
- Confirm user identity
- Gather symptoms and screenshots
- Perform basic troubleshooting
- Document all steps attempted
- Resolve known issues using knowledge base articles

## Escalation Criteria
A ticket should be escalated when:
- The issue cannot be resolved with documented Tier 1 steps.
- The issue requires administrator permissions.
- The issue impacts multiple users.
- The ticket is approaching SLA breach.
- The issue involves network infrastructure, server systems, or security access.

## Escalation Notes Template
```text
Issue Summary:
Steps Attempted:
User Impact:
Business Impact:
Screenshots/Logs:
Reason for Escalation:
```

## Escalation Example
A user reports that they cannot access the corporate VPN. Tier 1 verifies credentials, confirms internet connectivity, restarts the VPN client, and tests login again. The issue persists and may involve account permissions or VPN server access, so the ticket is escalated to Tier 2.
